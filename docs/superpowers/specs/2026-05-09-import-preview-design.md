# 导入预览功能设计方案

## 1. 概述

为场外基金收益计算器添加导入预览功能，用户在导入数据前可以清晰看到即将导入的内容，包括新增基金、已存在基金、新增交易记录、重复交易记录等统计信息，以及可以展开查看每条交易明细。

## 2. 交互流程

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│  选择JSON文件   │ ───▶ │   导入预览弹窗   │ ───▶ │   执行导入操作   │
│  (现有功能)     │      │   (新增)        │      │   合并/覆盖     │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

- **两步流程**：先预览，确认后执行
- **全部导入**：用户只需选择合并或覆盖，无需逐条选择
- **覆盖警告**：覆盖操作前显示醒目警告，需用户确认

## 5. 模块设计

### 5.1 ImportAppService 新增方法

| 方法 | 职责 |
|------|------|
| `analyzeImportData(data)` | 对比现有数据，分析导入内容的差异 |
| `getExistingFunds()` | 获取现有基金列表（供分析用） |
| `getExistingTrades()` | 获取现有交易列表（供分析用） |

### 5.2 ImportPreviewModal 新增模块

| 方法 | 职责 |
|------|------|
| `show(analysis)` | 渲染并显示预览弹窗 |
| `hide()` | 隐藏预览弹窗 |
| `bindEvents()` | 绑定展开/合并/覆盖等事件 |
| `_renderStatsCard()` | 渲染统计卡片区 |
| `_renderFundsList()` | 渲染基金列表区 |
| `_renderWarningBox()` | 渲染警告框 |
| `_toggleDetail()` | 切换交易明细展开/收起 |
| `_buildTradeRows()` | 生成交易明细HTML |

## 6. 样式规范

### 6.1 CSS 类名约定

```
.import-preview-modal    // 弹窗容器
.ip-stat-cards           // 统计卡片区
.ip-stat-card            // 单个统计卡片
.ip-stat-highlight       // 高亮卡片（新增）
.ip-stat-muted           // 弱化卡片（已存在/重复）
.ip-stock-section        // 基金列表区块
.ip-section-label        // 区块标题
.ip-stock-row            // 单个基金行
.ip-stock-row-header     // 基金行表头（可点击）
.ip-stock-detail         // 交易明细区
.ip-detail-table          // 交易明细表格
.ip-status-new           // 新增状态标签
.ip-status-dup           // 重复状态标签
.ip-bar-new              // 新基金指示条
.ip-bar-exists           // 已存在基金指示条
.ip-warning-box          // 警告框
```

### 6.2 颜色规范

继承现有 CSS 变量：
- 主题色：`--color-primary`
- 成功色（新增）：`--color-success` 或 `#4caf50`
- 警告色（重复）：`--color-muted` 或 `#999`
- 危险色（覆盖警告）：`--color-danger` 或 `#f44336`

## 7. 实现步骤

### 7.1 第一阶段：核心分析逻辑
1. 在 `ImportAppService` 新增 `analyzeImportData()` 方法
2. 实现基金比对逻辑（按 id 判断新增/已存在）
3. 实现交易比对逻辑（按 fundId + date + type + shares 判断重复）
4. 返回标准化的 `ImportAnalysis` 数据结构

### 7.2 第二阶段：预览弹窗UI
1. 在 `modal.js` 新增 `ImportPreviewModal` 弹窗HTML模板
2. 创建 `importPreviewModal.js` 模块，实现渲染和事件绑定
3. 实现统计卡片、基金列表、交易明细的HTML渲染
4. 实现展开/收起交互

### 7.3 第三阶段：集成和警告
1. 修改现有导入流程，调用预览弹窗
2. 实现覆盖警告确认逻辑
3. 对接执行导入逻辑（复用 `ImportAppService.importData()`）
4. 完成后刷新页面

## 8. 边界情况处理

| 情况 | 处理方式 |
|------|----------|
| 导入空数据 | 显示"没有可导入的数据"提示 |
| 文件格式错误 | 保持现有错误提示 |
| 全部重复（无新增） | 显示"全部重复，无需导入" |
| 单个基金大量交易 | 交易明细表格支持滚动 |
