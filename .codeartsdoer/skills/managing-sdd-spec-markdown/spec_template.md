# 需求规格文档模板

## 文档结构

### 1. 文档头部
```markdown
# 需求规格文档 - [功能名称]

**版本**: v1.0
**创建日期**: YYYY-MM-DD
**最后更新**: YYYY-MM-DD
**作者**: [作者名称]
**状态**: Draft/Review/Approved
```

### 2. 项目概述
```markdown
## 1. 项目概述

### 1.1 项目背景
[描述项目的业务背景和目标]

### 1.2 项目范围
[明确项目的功能范围和边界]

### 1.3 排除范围
[明确项目不包含的功能]
```

### 3. 功能需求
```markdown
## 2. 功能需求

### 2.1 [功能模块名称]

#### REQ-XXX: [需求标题]
**需求描述**:
[EARS格式的需求描述]

**验收标准**:
- Given [前置条件]
- When [触发动作]
- Then [预期结果]

**优先级**: High/Medium/Low
**相关模块**: [相关功能模块]
```

### 4. 非功能需求
```markdown
## 3. 非功能需求

### 3.1 性能需求
#### REQ-XXX: [性能需求标题]
**需求描述**: [EARS格式描述]
**验收标准**: [具体指标]

### 3.2 安全需求
### 3.3 可用性需求
### 3.4 兼容性需求
```

### 5. 数据需求
```markdown
## 4. 数据需求

### 4.1 输入数据
[描述系统需要的输入数据]

### 4.2 输出数据
[描述系统产生的输出数据]

```

### 6. 接口需求

```markdown
### 5.1 用户接口
```


## 编写指南

### 需求编号规则
- 功能需求: REQ-001 ~ REQ-999
- 非功能需求: NFR-001 ~ NFR-999
- 数据需求: DATA-001 ~ DATA-999
- 接口需求: INT-001 ~ INT-999

### 优先级定义
- **High**: 核心功能，必须实现
- **Medium**: 重要功能，应该实现
- **Low**: 辅助功能，可选实现

### 验收标准格式
使用 Given-When-Then 格式：
- **Given**: 初始状态/前置条件
- **When**: 触发动作/事件
- **Then**: 预期结果/后置条件

### 需求分组原则
1. 按功能模块分组
2. 相关需求集中描述
3. 依赖关系明确标注
4. 避免需求重复
