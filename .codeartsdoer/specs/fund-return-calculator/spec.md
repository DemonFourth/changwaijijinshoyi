## 1. 项目概述

### 1.1 项目背景
场外基金收益计算器是一个专门用于计算和管理场外基金（通过支付宝、天天基金等平台买卖的基金）投资收益的Web应用工具。该工具参考股票收益计算器的设计理念，但针对场外基金的特殊性进行适配，帮助投资者精确追踪基金投资收益、分析投资表现。

### 1.2 项目范围
本系统提供以下核心功能：
- **基金信息管理**：添加、编辑、删除基金基本信息（基金代码、名称）
- **基金数据获取**：通过基金API自动获取基金净值、名称等实时数据
- **交易记录管理**：记录买入、卖出、分红等交易操作
- **收益计算**：基于先进先出(FIFO)原则精确计算持仓成本和收益
- **数据可视化**：通过图表展示收益趋势、成本变化等
- **数据持久化**：本地存储交易数据，支持数据导入导出

### 1.3 排除范围
本系统**不包含**以下功能：
- 实时行情推送和价格预警
- 基金定投自动扣款管理
- 基金推荐和投资建议
- 多账户管理
- 与支付宝/天天基金等平台的数据同步

## 2. 功能需求

### 2.1 基金信息管理

#### REQ-001: 添加基金
**需求描述**:
When 用户点击"添加基金"按钮并输入基金代码, the system shall 自动获取基金名称并保存基金信息

**验收标准**:
- Given 用户在添加基金页面
- When 用户输入基金代码"519732"并点击保存
- Then 系统调用API获取基金名称"万家行业优选混合(LOF)"并保存基金信息

**优先级**: High
**相关模块**: 基金管理模块

#### REQ-002: 基金代码验证
**需求描述**:
If 用户输入的基金代码格式不正确或API查询失败, then the system shall 显示错误提示信息

**验收标准**:
- Given 用户在添加基金页面
- When 用户输入无效基金代码"123"
- Then 系统显示"基金代码格式错误，请输入6位数字"

**优先级**: High
**相关模块**: 基金管理模块

#### REQ-003: 编辑基金信息
**需求描述**:
When 用户点击基金卡片的编辑按钮, the system shall 允许用户修改基金名称并保存

**验收标准**:
- Given 用户已添加基金"519732"
- When 用户点击编辑按钮修改名称为"我的优选基金"并保存
- Then 系统更新基金名称为"我的优选基金"

**优先级**: Medium
**相关模块**: 基金管理模块

#### REQ-004: 删除基金
**需求描述**:
When 用户点击删除基金按钮并确认, the system shall 删除该基金及其所有交易记录

**验收标准**:
- Given 用户已添加基金"519732"且有交易记录
- When 用户点击删除按钮并确认
- Then 系统删除基金"519732"及其所有关联的交易记录

**优先级**: Medium
**相关模块**: 基金管理模块

### 2.2 基金数据获取

#### REQ-005: 获取基金实时数据
**需求描述**:
When 用户输入有效的基金代码, the system shall 从基金API获取基金名称和最新净值

**验收标准**:
- Given 用户输入基金代码"519732"
- When 系统调用基金API
- Then 系统正确解析GB2312编码的响应数据并显示基金名称和净值

**优先级**: High
**相关模块**: 数据服务模块

#### REQ-006: API编码处理
**需求描述**:
The system shall 正确处理基金API返回的GB2312编码数据

**验收标准**:
- Given 基金API返回GB2312编码的数据
- When 系统解析响应数据
- Then 基金名称正确显示为中文字符，无乱码

**优先级**: High
**相关模块**: 数据服务模块

#### REQ-007: API失败重试
**需求描述**:
If 基金API请求失败, then the system shall 重试最多3次并显示友好的错误提示

**验收标准**:
- Given 基金API服务不可用
- When 系统尝试获取基金数据
- Then 系统重试3次后显示"网络异常，请稍后重试"

**优先级**: Medium
**相关模块**: 数据服务模块

### 2.3 交易记录管理

#### REQ-008: 添加买入记录
**需求描述**:
When 用户填写买入表单（日期、金额、份额、手续费）并提交, the system shall 保存买入交易记录

**验收标准**:
- Given 用户选择基金"519732"
- When 用户输入买入日期"2024-01-15"、金额"10000元"、份额"800份"、手续费"0元"并提交
- Then 系统保存买入记录并更新持仓信息

**优先级**: High
**相关模块**: 交易管理模块

#### REQ-009: 添加卖出记录
**需求描述**:
When 用户填写卖出表单（日期、金额、份额、手续费）并提交, the system shall 保存卖出交易记录并计算收益

**验收标准**:
- Given 用户持有基金"519732"份额800份
- When 用户输入卖出日期"2024-06-15"、金额"12000元"、份额"800份"、手续费"0元"并提交
- Then 系统保存卖出记录并计算收益2000元

**优先级**: High
**相关模块**: 交易管理模块

#### REQ-010: 添加分红记录
**需求描述**:
When 用户填写分红表单（日期、分红金额、分红方式）并提交, the system shall 保存分红记录

**验收标准**:
- Given 用户持有基金"519732"
- When 用户输入分红日期"2024-03-15"、分红金额"200元"、分红方式"红利再投"并提交
- Then 系统保存分红记录并更新持仓信息

**优先级**: Medium
**相关模块**: 交易管理模块

#### REQ-011: 编辑交易记录
**需求描述**:
When 用户点击交易记录的编辑按钮修改信息并保存, the system shall 更新交易记录并重新计算收益

**验收标准**:
- Given 用户有买入记录"2024-01-15 买入10000元"
- When 用户修改金额为"12000元"并保存
- Then 系统更新交易记录并重新计算持仓成本和收益

**优先级**: Medium
**相关模块**: 交易管理模块

#### REQ-012: 删除交易记录
**需求描述**:
When 用户点击删除交易记录按钮并确认, the system shall 删除该交易记录并重新计算收益

**验收标准**:
- Given 用户有交易记录
- When 用户删除某条交易记录
- Then 系统删除该记录并重新计算持仓成本和收益

**优先级**: Medium
**相关模块**: 交易管理模块

### 2.4 收益计算

#### REQ-013: FIFO成本计算
**需求描述**:
The system shall 使用先进先出(FIFO)原则计算持仓成本和卖出收益

**验收标准**:
- Given 用户分两次买入：第一次1000份成本1.0元/份，第二次1000份成本1.2元/份
- When 用户卖出1000份
- Then 系统按第一次买入成本1.0元/份计算收益

**优先级**: High
**相关模块**: 计算模块

#### REQ-014: 持仓收益计算
**需求描述**:
The system shall 实时计算当前持仓的浮动盈亏和收益率

**验收标准**:
- Given 用户持有基金份额1000份，持仓成本10000元，当前净值12元/份
- When 系统计算持仓收益
- Then 显示持仓市值12000元，浮动盈亏2000元，收益率20%

**优先级**: High
**相关模块**: 计算模块

#### REQ-015: 已实现收益计算
**需求描述**:
The system shall 统计所有已卖出交易的累计收益

**验收标准**:
- Given 用户有3次卖出交易，收益分别为500元、800元、1200元
- When 系统计算已实现收益
- Then 显示累计已实现收益2500元

**优先级**: High
**相关模块**: 计算模块

#### REQ-016: 分红收益处理
**需求描述**:
The system shall 正确处理分红收益，区分现金分红和红利再投

**验收标准**:
- Given 用户收到现金分红200元
- When 系统计算收益
- Then 将200元计入已实现收益
- And 如果是红利再投，则增加持仓份额

**优先级**: Medium
**相关模块**: 计算模块

## 4. 数据需求

### 4.1 输入数据

#### DATA-001: 基金信息
- **基金代码**: 6位数字字符串（如"519732"）
- **基金名称**: 字符串（如"万家行业优选混合(LOF)"）
- **基金类型**: 字符串（如"混合型"）

#### DATA-002: 交易记录
- **交易日期**: 日期格式（YYYY-MM-DD）
- **交易类型**: 枚举值（买入/卖出/分红）
- **金额**: 数字（单位：元）
- **份额**: 数字（单位：份）
- **手续费**: 数字（单位：元）
- **备注**: 字符串（可选）

### 4.2 输出数据

#### DATA-003: 收益统计
- **持仓成本**: 数字（单位：元）
- **持仓市值**: 数字（单位：元）
- **浮动盈亏**: 数字（单位：元）
- **收益率**: 百分比数字
- **已实现收益**: 数字（单位：元）

### 4.3 数据约束

#### DATA-004: 业务规则约束
- 基金代码必须为6位数字
- 交易日期不能晚于当前日期
- 卖出份额不能超过持有份额
- 金额、份额、手续费必须为非负数

## 5. 接口需求

### 5.1 用户接口

#### INT-001: 页面布局
- **汇总页**: 显示所有基金列表和总体统计
- **详情页**: 显示单只基金的详细信息和交易记录
- **添加/编辑弹窗**: 用于添加和编辑基金、交易记录

#### INT-002: 交互方式
- 点击操作：添加、编辑、删除、查看详情
- 表单输入：基金代码、交易信息
- 图表交互：缩放、悬停显示详情

### 5.2 外部接口

#### INT-003: 基金数据API
- **接口地址**: http://fundgz.1234567.com.cn/js/{基金代码}.js
- **请求方法**: GET
- **响应格式**: JavaScript脚本（GB2312编码）
- **数据字段**: 基金代码、基金名称、单位净值、累计净值、净值日期

## 6. 约束条件

### 6.1 技术约束

#### CONS-001: 技术栈
- **前端**: 原生HTML + CSS + JavaScript（无框架）
- **图表库**: ECharts 5.x
- **数据存储**: localStorage
- **编码**: UTF-8（需处理API的GB2312编码）

#### CONS-002: 部署方式
- 纯静态部署，无需服务器端支持
- 可部署在任何静态文件服务器或直接本地打开

### 6.2 业务约束

#### CONS-003: 计算规则
- 使用FIFO原则计算持仓成本
- 分红收益单独统计
- 手续费计入成本

#### CONS-004: 数据范围
- 仅支持场外基金（开放式基金）
- 不支持场内基金（ETF、LOF等）
- 不支持货币基金和理财产品的特殊计算

## 7. 验收标准汇总

### 7.1 核心功能验收
1. 用户能够成功添加基金并自动获取基金信息
2. 用户能够添加、编辑、删除交易记录
3. 系统能够正确计算持仓成本和收益（FIFO原则）
4. 系统能够正确处理分红收益
5. 数据能够持久化存储并支持导入导出

### 7.2 性能验收
1. 所有操作响应时间 < 200ms
2. 页面加载时间 < 1秒
3. API请求超时时间 = 5秒

### 7.3 兼容性验收
1. 在Chrome、Firefox、Safari、Edge上正常运行
2. 在不同屏幕尺寸上自适应显示
